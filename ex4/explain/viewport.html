viewport란? 웹페이지가 사용자에게 보여지는 영역 휴대폰이나 태블릿이 나오기 이전의 웹페이지는 데스크탑 PC용을만 만들어서 고정된 크기를 가지고 있었으나, 현대에 와서는 화면이 작은 휴대폰이나 태블릿으로 보면 한 화면에 다보이지 않게 되었습니다. 그 이유는 대부분의 웹브라우저에서 웹페이지가 그려지는 렌더링 역역이 사용자에게 보여지는 영역인 뷰포트 보다 크기 때문입니다. 그래서 모바일 웹브라우저가 취하는 기본정책은 모두를 보여주기 위해 화면을 줄여서 보여주는 것입니다. 이러한 정책은 내용 전체를 볼 수는 있으나, 글자가 너무 작아져서 보기가 어려워지는 단점이 있습니다. HTML5에서 나온 viewport를 이용하게되면, 모바일 기기에서 실제 렌더링되는 영역과 뷰포트의 크기를 조절할 수 있습니다. 또한 줌 레벨도 조정을 할수 있습니다. 여기에서, - width=device-width : 페이지의 너비를 기기의 스크린 너비로 설정합니다. 즉, 렌더링 영역을 기기의 뷰포트의 크기와 같게 만들어 줍니다. (height도 width처럼 사용하여 정의가 가능합니다.) - initial-scale=1.0 : 처음 페이지 로딩시 확대/축소가 되지 않은 원래 크기를 사용하게 합니다. 0~10 사이의 값을 가지게 됩니다. - minimum-scale : 줄일 수 있는 최소 크기를 지정합니다. 0~10사이의 값을 가지게 됩니다. - maximum-scale : 늘릴 수 있는 최대 크기를 지정합니다. 0~10사이의 값을 가지게 됩니다. - user-scalable : yes or no 값을 가지며 사용자가 화면을 확대/축소 할 수 있는지를 지정합니다. - initial-scale : 초기 확대 값입니다. 1~10사이의 값을 가지게 됩니다. 다음의 조건들을 사용하여, 반응형 웹을 만들기 위해서는 뷰포트 메타 태그를 사용하여, CSS 를 통하여 이미지 사이즈나 스케일을 % 기준하여 조절을 해 주는것이 좋습니다.
